# ALine 下一阶段开发计划 Pro v2

## 阶段目标

Pro v2 面向 ALine 从“核心能力可用”走向“可持续发布、可扩展、可推广”。重点不再是单个功能堆叠，而是把软件体验、扩展生态和对外传播形成闭环。

## 方向一：软件优化

### 1. 稳定性与发布质量

目标：建立可重复、可追踪的发布质量门槛。

任务：

- 建立发布前检查清单：backend、UI、扩展加载、文档链接、示例项目、打包验证。
- 将 UI 测试拆成快速窄测与全量回归两层。
- 对项目打开、保存、扩展重载、图像数字化、处理流水线、图表导出建立冒烟测试。
- 梳理异常提示，确保用户看到的是可执行的错误说明，而不是 Python traceback。
- 给项目文件格式增加版本迁移记录和回滚保护。

验收：

- 每个发布候选版本都有一份测试摘要。
- 新建项目到导出图表的主流程可自动化验证。
- 项目文件升级路径有测试覆盖。

### 2. 性能与大数据体验

目标：让 ALine 能稳定处理科研常见的大曲线和高分辨率图片。

任务：

- 建立 10k、100k、1M 点曲线处理基准。
- 对重采样、平滑、FFT、多曲线对齐和绘图刷新做 profiling。
- 对自动取点与蒙版处理建立图片尺寸基准。
- 引入长任务进度、取消机制和后台执行队列。
- 对重复绘图、重复扩展扫描、重复项目树刷新做缓存或去抖优化。

验收：

- 大曲线处理有明确性能目标和回归曲线。
- 长耗时操作不阻塞主界面。
- 用户可取消自动取点、批量处理和大图导出。

### 3. UI 与工作流打磨

目标：把页面从“功能可达”提升到“工作流顺手”。

任务：

- 统一各页面空状态、错误状态、加载状态和完成状态。
- 优化项目树专注模式、全局资源、拖拽、重命名和批量操作。
- 优化数字化页：蒙版编辑、自动取点参数预览、取色/截图反馈、点列表编辑。
- 优化处理页：流水线可视化、步骤重排、参数复制、结果预览。
- 优化分析页：结果对比、报告模板预览、占位符搜索。
- 优化绘图页：样式预设、扩展叠加顺序、导出配置复用。

验收：

- 每个页面都有明确主任务和快捷路径。
- 常用操作不需要跨多处重复配置。
- UI 窄测覆盖核心交互。

### 4. AI 辅助能力

目标：让 AI 从“命令入口”升级为“项目内研究助手”。

任务：

- 明确 AI 可读取的项目上下文范围。
- 增加安全的只读分析工具：列出曲线、总结处理流水线、解释分析结果。
- 增加可确认的写操作：创建处理步骤、生成报告草稿、建议图表样式。
- 给 AI 操作添加预览、确认和撤销。
- 建立 AI 失败与超时提示机制。

验收：

- AI 不直接绕过项目管理器修改数据。
- 所有写操作可预览、可撤销。
- 用户能从 AI 面板完成至少 3 条典型工作流。

## 方向二：扩展社区

### 1. 扩展 SDK 与模板

目标：降低第三方扩展开发门槛。

任务：

- 建立 `extensions/templates/`，提供处理、分析、绘图、数字化四类模板。
- 提供扩展脚手架命令或脚本，生成文件、测试和 README。
- 提供本地扩展调试指南：重载、日志、断点、错误定位。
- 增加扩展元数据检查器，提前发现重复 type、错误 version、非法 field_type。
- 为 point-list line 协议提供可导入的测试 fixture。

验收：

- 新开发者 10 分钟内可创建并加载一个处理扩展。
- 扩展模板能通过最小测试。
- 文档示例与实际模板一致。

### 2. 扩展市场与分发

目标：让内置扩展、外部扩展和社区扩展可以被发现、安装、启停和升级。

任务：

- 定义扩展包结构：源码、manifest、README、图标、测试数据。
- 增加扩展来源：本地目录、压缩包、远程索引。
- 设计扩展详情页：简介、版本、作者、权限、参数字段、变更记录。
- 增加扩展版本检查和升级提示。
- 增加扩展启停状态导入/导出。

验收：

- 用户可以从本地包安装外部扩展。
- 扩展启停与配置可随项目或全局偏好保存。
- 远程索引格式稳定并有示例仓库。

### 3. 扩展质量与安全

目标：让扩展生态可控，而不是任意脚本堆积。

任务：

- 给扩展定义权限声明：文件读取、网络访问、外部进程、图像访问。
- 为外部扩展提供沙箱或受限运行策略调研。
- 增加扩展审查清单：协议、参数、异常、性能、文档、测试。
- 引入扩展评分维度：稳定性、适用领域、维护状态、示例数据。
- 对官方推荐扩展建立 CI 验证。

验收：

- 官方扩展都有测试和文档。
- 外部扩展加载失败不会影响核心应用启动。
- 用户能看到扩展权限和风险提示。

## 方向三：软件推广

### 1. 产品定位

目标：明确 ALine 的第一批目标用户和使用场景。

建议定位：

- 科研论文图表数据重建。
- 实验曲线数字化与清洗。
- 多曲线比较、分析和报告生成。
- 面向 Python 用户的可扩展科研图形工具。

任务：

- 梳理 3 个核心用户故事：论文图片取点、实验图像曲线提取、多曲线处理分析。
- 准备 3 个可公开示例项目。
- 准备功能矩阵，对比通用图像取点工具与脚本工作流。

验收：

- 首页 README 能在 1 分钟内说明 ALine 解决什么问题。
- 示例项目可复现完整流程。

### 2. 文档与教程

目标：让用户能独立完成第一次成功体验。

任务：

- 写一篇“5 分钟完成第一条曲线数字化”。
- 写一篇“从曲线处理到分析报告”。
- 写一篇“编写你的第一个 ALine 扩展”。
- 为每篇教程配套示例图片、示例项目和预期输出。
- 录制短视频或 GIF 展示关键流程。

验收：

- 新用户按教程可完成导入、校准、取点、处理、分析、导出。
- 扩展开发者按教程可创建一个外部扩展。

### 3. 发布与社区运营

目标：建立稳定反馈渠道和版本节奏。

任务：

- 准备 GitHub Release 模板：亮点、修复、破坏性变更、升级提示。
- 建立 issue 模板：bug、功能建议、扩展投稿、数据样例。
- 建立讨论区主题：使用问答、扩展分享、论文场景、路线图。
- 准备官网或文档站结构：快速开始、用户指南、扩展开发、API、路线图。
- 规划版本节奏：alpha、beta、稳定版。

验收：

- 每个版本有清晰 changelog。
- 用户反馈可分类、可追踪、可进入路线图。
- 社区扩展有投稿流程。

## 里程碑

### M1：Alpha 稳定版

- 主流程稳定。
- backend 与 UI 核心窄测通过。
- 扩展协议冻结。
- 文档覆盖用户首次体验和扩展开发。

### M2：Beta 生态版

- 外部扩展安装与管理可用。
- 扩展模板、扩展索引和官方示例扩展完善。
- 大数据性能有明确基线。

### M3：公开推广版

- 示例项目、教程、演示视频、发布说明齐备。
- 社区反馈渠道开放。
- 推荐扩展和贡献指南稳定。
