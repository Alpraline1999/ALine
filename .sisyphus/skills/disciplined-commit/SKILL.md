# Disciplined Commit

用于把“修改完成后如何提交”收口成稳定流程。

## 何时使用

- 用户要求“提交”“commit”“分批提交”
- 一项功能、修复、重构已形成可独立说明的切片
- 阶段性重构需要 `start` / `checkpoint` / `end` 节点提交
- 工作区有多类改动，需要拆成多个逻辑清晰的提交

## 不适用

- 直接相关验证还没通过
- 工作区混有大量无关改动且尚未分离
- 当前只是试验、探针、日志打印或方向未定
- 用户明确表示先不要提交

## 核心原则

1. 先验证，再提交。
2. 一次提交只表达一个目标。
3. 只暂存相关文件，不顺手清理无关内容。
4. 中文说明必须能让后来者看懂背景、修改、验证和剩余事项。

## 默认约定

- `feat:` / `fix:` / `refactor:` / `test:` / `docs:` / `chore:` 前缀优先。
- 标题应直接概括结果，不写空泛过程词。
- 有更窄验证时，不用“只看 diff”替代验证。
- 不使用 `git add -A`、`git commit -a`、交互式 git 控制台。
- 不把构建产物、缓存文件、测试副产物混入提交。

## 操作流程

### 1. 先界定提交边界

先用一句中文概括本次提交目标。若概括不清，先拆分改动。

对阶段性重构额外确认：
- 是否对应阶段文档中的检查点
- 是否已经形成最小闭环
- 是否混入下一阶段工作

### 2. 选择最贴近行为的验证

优先顺序：
1. 被改行为的最小验证
2. 被改模块的窄测
3. 相关 lint / 类型检查
4. 若确实没有可执行验证，再退到 diff 审查

规则：
- 验证失败时，不提交
- 修复后要复跑同一验证
- 影响面扩大时，主动补更广的回归

### 3. 审查工作区

明确区分：
- 本次任务改动
- 预存的无关改动
- 测试或运行带出的噪声文件

如果命令运行带脏了配置、缓存、构建产物，先恢复这些噪声，再继续提交。

### 4. 精确暂存

- 只 `git add <相关文件>`
- 不把无关文档、格式化、测试调整顺手带上
- 需要拆批次时，每一批都重新审视 `git diff --cached`

### 5. 组织提交信息

标题格式：

```text
<type>: 中文摘要
```

正文模板：

```text
阶段：
- Phase X / start|checkpoint|end

背景：
- 当前问题或目标

修改：
- 关键变更点

验证：
- 跑过的验证项

后续：
- 未进入本次提交的相邻工作
```

如果不是阶段性重构，可省略“阶段”段，但背景/修改/验证/后续仍应保留。

### 6. 提交前检查

提交前至少确认：
- `git status --short`
- `git diff --cached --stat`
- 必要时看 `git diff --cached`

如果暂存区范围和一句话目标不一致，先撤回本批重新整理。

### 7. 非交互式提交

使用：

```bash
git add <files>
git commit -m "标题" -m "正文"
```

如果是多批次提交，按“验证过的最小闭环”顺序提交，不按目录顺序机械切。

### 8. 提交后汇报

向用户明确说明：
- 提交 hash 和标题
- 本次提交覆盖的改动
- 跑过的验证
- 工作区是否还有剩余未提交改动

## 决策规则

### 什么时候必须拆成多个提交

- 一个提交里同时混有功能新增和无关修复
- 同时包含实现改动与大面积测试/文档改写，且两者可独立说明
- 工作区里存在多条互不依赖的改动线

### 什么时候可以合并成一个提交

- 测试、文档、实现紧密围绕同一目标
- 单独拆开会让任一提交失去可验证性或可理解性

## 输出要求

执行本技能时，最终应给出：
- 提交是否成功
- 新提交 hash
- 标题
- 验证摘要
- 剩余未提交改动

## 反模式

- “测试没过但先提交保存一下”
- “顺手把别人的改动也带上”
- “工作区很脏，直接 `git add -A`”
- “标题只有阶段号，没有实际内容”
- “只说已提交，不说验证和剩余改动”

## 示例

```text
refactor: 阶段22-开始 提取 TreeManager 到独立模块

阶段：
- Phase 22 / start

背景：
- ProjectManager 体量过大，项目树职责耦合严重

修改：
- 提取 TreeManager 类
- 调整主调度入口的依赖注入

验证：
- 项目树展开/折叠窄测通过
- 架构护栏测试通过

后续：
- 下一节点继续拆 DataFileManager
```
