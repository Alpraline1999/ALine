# 优化 03：处理页统一切换为短提示并抽象为方法

## 问题描述

处理页在执行 pipeline 时存在多种大段报错场景：

1. **多曲线扩展**在重采样后仍然报错需要重采样（预检查 vs 运行时验证不一致）
2. **其他处理扩展**的 ValueError 也被 `_on_async_pipeline_error` 和同步路径以"处理错误:"前缀显示红色大字
3. 大段报错包含完整 traceback，信息过载

用户要求：将所有处理扩展报错统一为红色小段文字提示，并抽象为通用方法以便复用。

## 方案

1. 在 `process_page.py` 中抽象 `_set_pipeline_hint` 方法：
   - 统一接管所有处理扩展的报错显示
   - 自动截断 traceback，提取首行有用消息
   - 标记为红色样式但不显示"处理错误:"前缀

2. 同步路径 `_run_pipeline_now` 也切换到同一方法

3. 保留非处理扩展的错误（如保存失败、导出失败）仍使用 InfoBar.error
