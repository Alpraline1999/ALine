# Phase 32：分析扩展 bool 运行时加固

## 目标与完成定义

**目标**：彻底收口分析扩展运行链路中的 bool 输入/输出污染，修复“运行分析”仍报 `"bool" object is not iterable` 的残余路径。

**完成定义**：
- 数据分析页所有分析扩展点击“运行分析”不再抛出该错误
- 内置分析、外部分析、结果保存、结果预览、报告渲染路径全部通过统一防御层
- 至少有针对 UI 入口和运行时入口的回归测试

## 当前问题

- 现有修复已经覆盖部分 `list(True)` / `iter(True)` 路径，但用户验收表明分析页仍存在残余入口
- 风险不只在 handler 输入，也可能在：
  - result dict 的任意字段
  - 页面层二次归一化
  - 结果 tab 恢复、报告渲染、保存结果快照

## 优化方案

### 1. 统一 bool 防御策略

- 对“期望 list-like”的字段统一走 `_ensure_list()` / 等效 helper
- 对“期望 line-like”的字段统一走 `normalize_line()` / `normalize_lines()`
- 明确 bool 一律视为空值，而不是隐式 `int(True) == 1`

### 2. 收口分析页所有渲染入口

- 运行分析后首次渲染
- 历史结果 tab 切换
- 结果摘要表
- 报告模板预览
- 保存/导出结果时的结果重建

### 3. 用测试锁住回归

- handler 返回 bool 字段
- params 中混入 bool
- saved result / reopened result 中包含 bool 污染字段

## 验收要点

- 所有分析扩展点击“运行分析”均不再报 `"bool" object is not iterable`
- 结果图表、表格、报告预览都能优雅降级
- 回归测试覆盖运行时与页面层

## 边界与约束

- 不再兼容明显非法的旧分析结果协议
- 不把 bool 隐式转换为有效曲线索引或有效数据点
