# Phase 34：节点备注与节点管理信息面板

## 目标与完成定义

**目标**：为项目树节点增加可编辑备注，并在节点管理面板中把备注作为主描述信息展示，替换当前泛化的“数据文件、系列、图像...”说明。

**完成定义**：
- 节点支持备注字段，项目保存/加载后可保留
- 节点管理面板优先显示备注
- 右键菜单支持“设置/修改备注”
- 双击节点管理面板中的描述区域可弹窗编辑备注

## 当前问题

- 当前节点管理面板展示的是通用类别说明，信息密度低
- 用户无法在树层对节点补充实验语义、处理说明或上下文备注

## 优化方案

### 1. 节点数据模型增加 remark

- remark 属于用户可编辑元数据
- 适用于 folder、data_file、series、image_work、curve、analysis_result 等主要节点

### 2. 节点管理面板语义调整

- 备注存在时显示备注
- 备注为空时显示“未设置备注”
- 原有通用类别说明退居次要信息或固定标签

### 3. 两个编辑入口

- 项目树右键菜单
- 节点管理面板描述区双击

## 验收要点

- 设置备注后树刷新、面板刷新、页面切换后都能看到一致结果
- 备注修改不会影响节点名称、ID 和业务引用
- 未设置备注时空状态文案清晰

## 边界与约束

- 本阶段不引入富文本备注
- 备注不参与节点唯一性判断
