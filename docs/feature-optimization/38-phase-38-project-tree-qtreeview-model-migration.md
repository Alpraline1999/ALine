# Phase 38：项目树 QTreeView / QAbstractItemModel 迁移

## 目标与完成定义

**目标**：真正完成项目树从 `QTreeWidget` 到 `QTreeView + QAbstractItemModel` 的迁移，并保持现有视觉样式和交互能力不回退。

**完成定义**：
- 项目树底层改为 model/view 架构
- 样式、缩进、换行、图标、tooltip 与当前版本保持一致或等效
- 拖放、右键菜单、专注模式、懒加载和多选能力保持可用
- 500+ 节点场景下展开/滚动性能优于当前实现

## 当前问题

- 现有树虽然已做了一部分懒加载和 helper 拆分，但仍深度依赖 `QTreeWidgetItem`
- 现有逻辑难以继续优化到真正的虚拟滚动和模型级过滤

## 设计原则

### 1. 样式不作为回退项

- 迁移目标不是“换技术栈顺便改视觉”
- 当前用户已形成使用习惯，样式、图标和操作语义应尽量保持不变

### 2. 先分离数据模型，再迁移 view

- 先抽出 tree domain adapter / index mapping
- 再实现 `QAbstractItemModel`
- 最后替换 view、delegate 和 action 路由

### 3. 功能等价优先于一次性扩展

- 先保证现有功能不退化
- 性能收益作为迁移后的直接验收项

## 验收要点

- 当前共享项目树所有页面都能正常工作
- 节点选择、激活、拖放、批量操作、右键菜单全部可用
- 专注模式和全局资源过滤仍正确
- 长名称换行和 tooltip 行为无明显退化

## 边界与约束

- 本阶段不重新设计项目树信息架构
- 不把迁移与新的节点能力绑定在同一个提交面中
