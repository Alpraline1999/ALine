# Phase 37：设置页语言切换与运行时国际化

## 目标与完成定义

**目标**：在 Phase 28 的 gettext 基础上，真正让用户可以在设置页切换语言，并让应用按配置加载语言资源。

**完成定义**：
- 设置页新增语言选择项
- 语言偏好可持久化
- 应用启动时按用户配置加载语言
- 切换语言后能明确提示刷新/重启策略

## 当前代码现状

- `core/i18n.py` 已有基础 gettext 初始化
- 当前语言固定为 `zh_CN`
- 设置页还没有语言选项，也没有 locale 持久化配置

## 优化方案

### 1. locale 配置与运行时加载

- 增加语言配置源，例如 `zh_CN` / `en_US`
- `core/i18n.py` 根据配置而不是硬编码值初始化

### 2. 设置页语言入口

- 在设置页“外观”或“通用”分组加入语言选择
- 明确说明是“即时刷新”还是“重启后生效”

### 3. 构建与资源流程补齐

- `.pot` / `.po` / `.mo` 流程补齐
- 打包时确保 locale 资源进入发行物

## 验收要点

- 用户可在设置页看到并保存语言偏好
- 重启应用后语言配置生效
- 缺失翻译时能优雅回退到源码字符串

## 边界与约束

- 本阶段只要求中文和英文的运行时框架
- 不要求一次性完成所有页面的高质量英文翻译
