# Phase 7：全局模板/预设/扩展配置资产库优化

## 目标与完成定义

目标：

- 把 `global_assets` 承载的多类全局资源，从“能存取”提升到“可复用、可预览、可迁移、可理解”的资产工作流。

完成定义：

- 模板、主题、样式、扩展配置的使用体验更统一。
- 用户能更容易理解某项全局资产会影响哪里、如何复用和迁移。

## 当前审查结论

当前 `global_assets` 已经承载：

- SavedPipeline
- FigureTemplate
- ReportTemplate
- CurveStyleTemplate
- PlotTheme
- ExtensionConfigPreset
- AI prompt / skill / agent

这说明底层资产模型已经很丰富，但用户工作流仍偏基础：

- 列表
- 新建/删除/更新
- 个别页面直接加载使用

## 主要问题

### 1. 资产管理入口分散

影响：

- settings/process/analysis/chart 各自只看到局部资源。
- 用户不容易形成“这是同一套全局资产库”的心智。

### 2. 预览和应用路径不够统一

影响：

- 用户可能知道资源存在，但不确定它会作用于：
  - 当前页面
  - 当前对象
  - 全局默认
  - 后续新建对象

### 3. 迁移与复用能力不足

影响：

- 复制、导入/导出、另存为、默认值切换、依赖提示等还没有形成统一体验。

## 推荐优化方向

### A. 统一资产类型的基础操作语义

建议对主要全局资产统一支持以下动作语义：

- 预览
- 复制
- 另存为
- 导入/导出
- 设为默认
- 查看影响范围

### B. 加强“应用到哪里”的可解释性

建议清晰区分：

- 作为当前页面临时应用
- 作为当前对象模板应用
- 作为默认模板/默认配置
- 作为全局库项保存

### C. 扩展配置资产工作流优化

尤其建议加强：

- 一个扩展多个配置 preset 的管理
- 默认 preset 切换
- preset 与扩展版本/能力的兼容提示

### D. 资源依赖与引用提示

建议在轻量层面做到：

- 当前 pipeline / report / plot theme 是否正在被使用
- 删除前提示引用风险
- 复制后避免名称冲突和误覆盖

## 验收标准

- 用户可以更容易预览、复用和迁移全局资产。
- 不同类型的全局资产在基本操作语义上更一致。
- 扩展配置、模板和主题的“应用范围”更清晰。

## 本阶段不纳入的范围

- 云同步资产库
- 在线资源市场
- 权限协作系统

## 风险与回退

风险：

- 若所有资产类型都要求一步到位完全统一，范围会过大。
- 若“默认值”和“当前应用值”语义不区分，容易误导用户。

回退方式：

- 先统一操作语义和提示口径。
- 先覆盖 plot/process/analysis 最常用的资产类型，再逐步扩展到 AI 资源等次级对象。
