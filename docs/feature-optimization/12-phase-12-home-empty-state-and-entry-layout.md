# Phase 12：首页空状态布局与入口区紧凑化

## 目标与完成定义

目标：

- 修正首页在“最近项目为空”时的布局松散问题，让按钮区、最近项目标题与空状态提示保持靠上、紧凑、稳定的视觉结构。
- 保持“有最近项目”和“无最近项目”两种状态下的布局切换可预测，不引入额外跳动和样式分叉。

完成定义：

- 当 `load_recent()` 为空时，首页按钮区、`最近项目` 标题、`暂无最近项目` 提示保持上对齐紧凑布局，不再出现大面积空白被中间内容撑开的情况。
- 当存在最近项目时，列表区域仍可按原设计占据剩余高度并滚动，不影响当前列表体验。
- 空状态与非空状态的布局切换由统一 helper 管理，而不是分散在 `setup_ui()` / `refresh_recent()` 中硬编码控制。
- 增加针对空状态布局的窄测试，避免后续样式调整再次把页面拉回“默认拉伸布局”。

## 当前审查结论

当前 `HomePage` 的问题不是单一样式值不对，而是布局策略没有区分数据状态。

已确认现状：

- `ui/pages/home_page.py`
  - `_recent_scroll` 在初始化时固定为 `Expanding`，并通过 `layout.addWidget(self._recent_scroll, 1)` 持续占据剩余高度。
  - `_no_recent` 只是简单 `show()/hide()`，没有联动调整内容区拉伸策略。
  - `refresh_recent()` 在空状态时只隐藏 scroll 区，不会切换占位布局模式。
- `tests/pages/test_home_page.py`
  - 当前只有“最近项目滚动区应占剩余高度”的正向测试。
  - 尚未覆盖“最近项目为空时应切换为顶部紧凑布局”的反向布局约束。

这意味着当前页面默认永远按“有滚动列表”的版式搭建，空状态只是把列表藏起来，因此视觉上会留下错误的纵向留白。

## 主要问题

### 1. 空状态与非空状态共用同一拉伸布局

影响：

- 空状态下按钮区和提示文案虽然顺序正确，但页面主体仍按“列表填满”思路排版。
- 后续再加首页入口或说明文案时，这种问题还会继续放大。

### 2. 数据状态切换缺少单一布局控制点

影响：

- 当前靠 `show()/hide()` 切换控件显隐，不能保证拉伸、边距、间距一起切换。
- 后续主题或首页入口继续微调时，容易在 `setup_ui()` 和 `refresh_recent()` 两侧出现重复修补。

### 3. 缺少空状态布局回归测试

影响：

- 现有测试只验证“有数据时 scroll 应该填满剩余空间”，没有守住“无数据时必须收紧”的另一半约束。
- 首页视觉问题容易在后续 UI 抛光中悄悄回归。

## 推荐优化方向

### A. 为最近项目区建立显式布局模式

建议引入单一 helper，例如：

- `_set_recent_section_mode(has_recent: bool)`

它至少统一控制：

- `_no_recent` 的显隐
- `_recent_scroll` 的显隐
- `_recent_scroll` 的 `QSizePolicy`
- 内容布局中最近项目区域的 stretch 策略

目标是把“空状态布局”和“列表布局”定义为两个明确模式，而不是依赖控件是否隐藏的副作用。

### B. 空状态使用顶部紧凑布局，非空状态保留滚动填充

建议策略：

- 空状态：
  - 按钮区、标题、占位文案都保持顶部堆叠
  - 最近项目区域不再申请额外伸展
  - 允许在底部保留自然留白，但不让中间控件被纵向撑散
- 非空状态：
  - 保持当前 scroll 区填充剩余高度
  - 列表仍支持滚动和后续增长

### C. 避免把空状态逻辑散落到多个位置

建议边界：

- `setup_ui()` 只创建结构，不决定最终模式
- `refresh_recent()` 只负责读数据并切换模式
- 如果后续首页继续增加 onboarding 或快捷入口，统一通过最近项目区模式来协调，不新增新的局部 patch

### D. 补充首页空状态窄测试

建议优先增加：

- `load_recent()` 为空时：
  - `_no_recent.isHidden()` 为 `False`
  - `_recent_scroll.isHidden()` 为 `True`
  - 最近项目区域不再占据剩余拉伸空间
  - 标题和占位提示保持靠上，垂直间距受控
- `load_recent()` 非空时：
  - 保持现有 scroll 扩展行为

测试方式以 `offscreen` / widget geometry 检查为主，不做重型截图测试。

## 验收标准

- 首页在无最近项目时呈现靠上紧凑布局。
- 首页在有最近项目时保持滚动填充体验，不引入列表回归。
- 布局模式切换由单一 helper 管理。
- 新增空状态布局窄测试并稳定通过。

## 本阶段不纳入的范围

- 首页 banner 视觉重做
- 最近项目卡片化改版
- 首页功能入口增减
- 主页整页响应式重设计

## 风险与回退

风险：

- 如果直接修改现有 `QVBoxLayout` stretch 而不封装模式切换，后续继续加首页内容时仍会再次失控。
- 如果用固定高度强行压紧空状态，可能损伤不同窗口尺寸下的适应性。

回退方式：

- 先用显式模式切换收口布局策略。
- 若几何验收不稳定，优先回退到“拉伸策略 + sizePolicy”控制，不走硬编码高度方案。
