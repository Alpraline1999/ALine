# Phase 17 Task 6: 为多选专注补显式选择快照

## 目标

修复“专注所选”仍依赖菜单关闭后即时 selection 状态的问题，让多选专注在右键菜单、项目树管理对话框与树作用域动作中统一使用显式选中快照。

## 实施

1. 新增显式专注入口
   - 为项目树提供按节点 key 集合直接进入专注的能力
   - 不再要求右键菜单回调重新从当前 selection 推断目标
2. 菜单与对话框统一改用快照
   - 右键菜单构建时捕获选中节点 key 集合
   - 管理对话框按钮切换也使用同一套显式专注入口
3. 补充窄测试
   - 覆盖 selection 已变化时仍可按快照进入多选专注
   - 覆盖菜单/对话框路径对多选专注的一致性
