# Phase 10 Task 1: 静态未定义名称守护栏 + 关键 UI 烟测

## 目标

把这轮已暴露的 `NameError` / 旧符号残留 / 回调注入缺失问题收口成可重复执行的轻量守护栏，避免后续优化阶段继续被同类低层错误打断。

## 实施

1. 扩展 `scripts/structure_check.py`
   - 增加 `F821` 未定义名称检查入口
   - 增加关键 UI 导入烟测入口
   - 保持现有结构检查项不被破坏
2. 增加 Phase 10 相关窄测
   - `MainWindow()` / `SettingsPage()` / `ProjectTreeWidget()` 的 `offscreen` 导入或构造烟测
   - 保存项目路径整理 `_safe_filename()` 的直接行为测试
   - 扩展配置导出 / 设为默认等高风险入口的最小行为测试
3. 清理守护栏发现的确定性问题
   - 漏导入
   - support/helper 拆分后残留的失效符号
   - 右键菜单或回调注入遗漏
4. 产出阶段收尾文档
   - 记录本阶段新增的守护栏范围
   - 记录仍未纳入的重型检查项
