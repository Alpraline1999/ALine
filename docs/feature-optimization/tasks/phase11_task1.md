# Phase 11 Task 1: ChartPage 主题刷新路径切分 + 轻量观测点

## 目标

优先收口 `chart_page.py` 中最影响体验的主题切换与大曲线重绘路径，使“主题刷新”和“数据/样式重绘”不再完全走同一条同步入口，并补上最小性能观测点。

## 实施

1. 梳理 `ChartPage` 重绘入口
   - 区分主题切换触发的刷新
   - 区分数据、样式、扩展变更触发的重绘
   - 避免主题切换直接走无差别完整重绘入口
2. 为大曲线路径补强阶段化行为
   - 主题切换时优先沿用调度入口，而不是直接 `_redraw_now()`
   - 对大曲线保留 decimated 预览 + 延迟完整重绘
   - 对过期主题刷新或重绘请求保留最新请求
3. 增加轻量观测点
   - 记录最近一次重绘原因
   - 记录最近一次重绘耗时 / 点数摘要
   - 提供可测试的状态字段或 helper，而不是仅靠人工体感
4. 增加 Phase 11 窄测
   - 主题刷新走调度路径的测试
   - 大曲线重绘路径或摘要状态的测试
5. 产出阶段收尾文档
   - 记录本阶段对 `ChartPage` 的路径切分结果
   - 明确 `DataPage` / `DigitizePage` 等后续拆分边界仍留待下一轮
