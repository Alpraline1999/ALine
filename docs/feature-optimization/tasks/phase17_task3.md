# Phase 17 Task 3: 修复树作用域清理的项目上下文与 root-scoped 判定

## 目标

修复项目树“清理空文件夹”在真实 UI 路径中偶发误判为“无需清理”的问题，确保树作用域入口始终对准当前项目上下文，而不是停留在过期的全局当前项目。

## 实施

1. 收口树作用域清理上下文
   - 检查树作用域菜单、项目根节点菜单和项目树管理对话框的项目激活时机
   - 确保 root-scoped 清理前使用当前树上下文对应的项目
2. 保持 root-scoped 与子树清理语义区分
   - root-scoped 清理继续作用于当前项目整棵树
   - 子树清理继续只作用于指定文件夹子树
3. 补充窄测试
   - 覆盖树作用域清理的项目激活
   - 覆盖 root-scoped 清理与“清理空子文件夹”在反馈和刷新上的差异
