# Phase 18 Task 1: 抽取未保存项目关闭确认的三态契约

## 目标

把未保存项目关闭流程从二选一确认升级为三态确认，统一输出 `保存 / 不保存 / 取消`，避免主窗口、项目树和数字化页各自维护不同的关闭逻辑。

## 实施

1. 新增共享关闭确认对话框
   - 在 `ui/dialogs/` 下创建共享 helper
   - 让对话框显式返回保存、放弃和取消三种结果
2. 收口项目关闭入口
   - `ui/main_window.py`
   - `ui/widgets/project_tree.py`
   - `ui/pages/digitize_page.py`
3. 保持关闭副作用可控
   - 只有保存或放弃时才真正关闭项目
   - 取消时保留当前项目与页面状态不变
