# Phase 16 Task 2: 统一项目树 folder 能力与批量动作契约

## 目标

收口项目树根分组、普通子文件夹、项目根节点、批量操作、拖放移动和 DataPage 节点管理之间的语义漂移。

## 实施

1. 统一 folder 语义
   - 普通子文件夹使用真实节点名，不再显示父分组名称
   - 仅根分组 folder 保持受保护语义
2. 扩展 folder 动作矩阵
   - 普通子文件夹补齐重命名、删除、移动
   - 根节点菜单补齐保存项目、另存项目、关闭项目
3. 扩展批量与拖放能力
   - 放宽批量删除/移动的 payload 约束到同类节点集合
   - 让 `source_file` 在源文件集合内支持节点移动，同时保留导入语义
4. 修复空文件夹清理入口
   - 确保 `root_id=None` 的全局清理入口可用
5. 补充窄测试
   - 覆盖 folder 节点菜单与项目根菜单动作
   - 覆盖普通子文件夹名称显示和批量/拖放行为
   - 覆盖清理空文件夹根级入口
