# Phase 17 Task 1: 修复根级清理空文件夹契约

## 目标

修复项目树树作用域入口的“清理空文件夹”行为，确保它能稳定清理当前项目中的空普通文件夹，同时继续保留“清理空子文件夹”的子树语义。

## 实施

1. 梳理 root-scoped 清理入口
   - 检查项目树树作用域菜单、项目根节点菜单和项目树管理对话框的调用路径
   - 确认它们都统一走 `ProjectTreeCommandService.prune_empty_folders(root_id=None)`
2. 收口清理作用域
   - 明确 root-scoped 清理与子树清理的参数差异
   - 修复 UI 层调用或文案层可能导致的误判
3. 补充窄测试
   - 覆盖 `root_id=None` 的清理行为
   - 覆盖树作用域清理后的刷新与反馈
