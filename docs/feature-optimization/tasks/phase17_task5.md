# Phase 17 Task 5: 为根级清理补显式项目作用域

## 目标

修复“清理空文件夹”仍依赖隐式 current project 的问题，让根级清理在菜单与树作用域路径中使用显式项目上下文。

## 实施

1. 收口项目作用域解析
   - 为项目树根级清理补充显式 `project_id` 解析路径
   - 优先使用右键锚点、当前选择或唯一项目来确定清理目标
2. 避免依赖过期全局状态
   - 不再仅依赖 `project_manager.current_project`
   - 在执行清理前切换到解析出的目标项目
3. 补充窄测试
   - 覆盖 stale current project 下的根级清理
   - 覆盖带显式项目作用域的树作用域入口
