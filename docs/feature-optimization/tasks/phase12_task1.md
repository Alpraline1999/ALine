# Phase 12 Task 1: 首页最近项目空状态布局模式收口

## 目标

把首页“最近项目为空”和“最近项目非空”两种布局模式收口到单一 helper，避免空状态仍沿用滚动列表的拉伸策略。

## 实施

1. 为最近项目区增加显式布局模式 helper
   - 统一控制空状态与非空状态的显隐
   - 统一控制最近项目滚动区的 stretch / sizePolicy
   - 避免把布局切换逻辑散落到 `setup_ui()` 和 `refresh_recent()`
2. 调整首页初始化与刷新流程
   - 初始化时只负责搭建结构
   - 刷新时只负责读取最近项目并切换布局模式
   - 保持有数据时列表继续占据剩余高度并可滚动
3. 增加 Phase 12 窄测试
   - 空状态下最近项目区应保持顶部紧凑
   - 非空状态下最近项目区仍保持滚动扩展
4. 产出阶段收尾文档
   - 记录本阶段完成内容和剩余边界
