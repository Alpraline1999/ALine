# Phase 17 Task 4: 修复多选专注入口与刷新恢复

## 目标

修复项目树多选专注在真实 UI 路径中无法稳定进入的问题，保证右键菜单、项目树管理对话框和树刷新后的选择恢复都支持多选专注。

## 实施

1. 保留右键上下文中的多选状态
   - 检查项目树右键菜单是否在锚点不命中已选项时错误清空多选
   - 保持上下文菜单对已存在多选的识别
2. 修复刷新后的专注恢复
   - 让树刷新过程能正确保留多选焦点集合与已选节点
   - 避免 refresh 后把多选专注降级成单选或丢失选择
3. 补充窄测试
   - 覆盖右键菜单触发时的多选保留
   - 覆盖 refresh 后多选专注与当前选择恢复
