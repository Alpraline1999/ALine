# Phase 3：可视化工作流与反馈增强

## 目标与完成定义

目标：

- 在 Phase 1 和 Phase 2 的基础上，进一步提升可视化页的可操作性和可解释性。
- 让“曲线可见性”和“扩展影响范围”对用户更透明。

完成定义：

- 用户可以更容易恢复被隐藏的曲线集合。
- 用户可以直接看到当前扩展影响哪条曲线、哪些样式字段。
- 图表页的交互提示能帮助定位冲突，而不是让用户靠猜。

## 需要优化的功能列表

### 1. 可见性恢复类动作

建议新增：

- 全部显示
- 反选显示状态
- 恢复上一次“仅显示选中”前的可见性快照

原因：

- 当前已有“仅显示选中”和批量隐藏/显示，但恢复路径不够直观。

### 2. 扩展目标曲线提示

建议新增：

- 在扩展状态区显示：
  - 目标曲线名称
  - 目标曲线来源路径
  - 当前实例影响字段数

原因：

- 当前扩展目标解析有 fallback 逻辑，用户需要更直接的反馈。

### 3. 样式覆盖提示

建议新增：

- 在曲线样式页和绘图样式页提供轻量提示：
  - 当前字段是否被扩展接管
  - 若被接管，由哪个扩展接管
- 对关键字段显示只读/受控提示，而不是让用户误以为 UI 失效。

### 4. 扩展影响摘要

建议新增：

- 在绘图扩展列表中提供“影响摘要”文本，例如：
  - 修改颜色/线宽
  - 修改图例/网格
  - 直接后处理 artist

原因：

- 当前用户只能看到扩展类型和参数，不容易预判结果。

## 推荐方案

### 方案一：轻量提示优先

推荐先做。

内容：

- 不新增复杂面板。
- 利用现有 hint label / tooltip / extension panel 状态区补充来源与影响信息。

适用：

- 低风险、快速提升可解释性。

### 方案二：显式样式来源面板

后续可选。

内容：

- 新增一个“样式来源/冲突检查”区域，列出当前选中曲线的关键字段来源。

适用：

- 在 Phase 2 完成 authority 契约后再考虑。

## 验收标准

- 用户执行“仅显示选中”后，能一键恢复完整视图。
- 选中任意扩展实例后，能看到它作用于哪条曲线、影响哪些样式层。
- 当样式面板字段被扩展接管时，UI 不再表现为“可编辑但改了无效”。

## 本阶段不纳入的范围

- 新增大型可视化分析功能
- 重做 chart page 整体布局
- 扩展市场或扩展安装流程优化

## 风险与回退

风险：

- 如果在 authority 契约稳定前就做大量提示，后续文本容易失效。
- 可见性快照若设计成隐式全局状态，可能与图片快照/绘图快照语义混淆。

回退方式：

- 先落“全部显示”和扩展目标提示，不强上历史快照恢复。
- 先做只读提示文案，不立即引入复杂来源面板。
