# Phase 36：专注模式下的全局资源节点对齐

## 目标与完成定义

**目标**：在专注模式下，项目树“全局资源”节点按页面显示必要内容，保证用户在聚焦当前工作时仍能访问直接相关的模板、样式和扩展配置。

**完成定义**：
- 数据处理页显示 `Pipelines`、`扩展配置/处理扩展`
- 数据可视化页显示 `曲线样式`、`绘图样式`、`扩展配置/绘图扩展`
- 数据分析页显示 `报告模板`、`扩展配置/分析扩展`
- 数字化页显示 `扩展配置/数字化扩展`

## 当前问题

- 专注模式目前主要收缩业务节点，但“全局资源”子树未按页面能力精细裁剪
- 结果是用户在专注模式下丢失当前页仍然需要的全局资产入口

## 优化方案

### 1. 定义页面到全局资源的映射矩阵

- 每个工作页声明自己在专注模式下允许暴露的全局资源类型
- 保持映射配置集中管理，避免散落在页面逻辑中

### 2. 项目树过滤链路扩展

- 专注模式过滤时，不只过滤业务节点，也过滤全局资源节点
- 允许只显示必要分组，而不是整个 `global_root`

## 验收要点

- 专注模式下仍然只看到当前工作直接相关内容
- 不同页面切换时，全局资源子树同步切换
- 非专注模式行为完全不变

## 边界与约束

- 本阶段不调整全局资源的整体分组结构
- 只改专注模式下的可见性矩阵
