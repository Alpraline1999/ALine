# Phase 1：AppShell 与运行时边界

## 目标与完成定义

目标：

- 把主窗口重建为壳层。
- 建立统一的 `AppContext`、命令模型、事件模型。
- 让共享项目树通过边界协作，而不是直接操纵页面和全局状态。

完成定义：

- `MainWindow` 只负责装配、导航、路由和订阅。
- 跨页协作只通过显式命令和事件。
- 共享树具备 `View + Builder + Dispatcher` 三段边界。

## 进入前提

- `Phase 0` 完成。
- 已固定依赖禁令与架构测试方向。

## 本阶段纳入的状态与边界

- 纳入：
  - `AppContext`
  - `AppCommand`
  - `TreeCommand`
  - `SessionEvent`
  - 共享树的视图、构建、动作分发边界
- 暂不迁移页面内部完整业务状态，只为后续迁移打边界。

## 本阶段禁止改动的区域

- 禁止提前重写页面业务算法。
- 禁止直接拆 `ProjectManager` 内部逻辑。
- 禁止在页面之外新增第二套树动作入口。

## 目标接口/类型/运行时对象

- `AppContext`
  - 持有 `project_session`、`asset_catalog`、`extension_runtime`、`event_bus`
- `NodeRef`
  - 用于统一表示共享树节点与虚拟叶节点
- `TreeCommand`
  - 表示树上的双击、右键动作、拖放动作
- `AppCommand`
  - 表示壳层可分发的全局动作
- `SessionEvent`
  - 表示项目、资产、页面状态需要感知的系统事件

## 实施顺序

1. 建立 `AppContext` 与基础消息模型。
2. 调整 `MainWindow`，移除对页面私有方法和私有刷新函数的直接访问。
3. 统一页面公开入口：
   - 选择事件入口
   - 激活动作入口
   - AppCommand 入口
   - SessionEvent 刷新入口
4. 拆共享树：
   - `ProjectTreeView`
   - `ProjectTreeBuilder`
   - `ProjectTreeActionDispatcher`
5. 让树和页面都通过 `NodeRef` 与命令协作，而不是直接共享内部细节。

## 兼容/迁移策略

- 保留现有行为结果，但不保留旧的私有方法调用路径。
- 任何旧刷新链如果仍存在，只允许作为短期适配层，不允许新增调用点。

## 验收标准

- `MainWindow` 中不再直接调用页面私有方法。
- 共享树不再直接修改项目数据或页面内部对象。
- 新的跨页交互都能归约为命令和事件。

## 提交检查点

- 检查点 1：`AppContext` 与基础命令/事件模型落地。
- 检查点 2：`MainWindow` 壳层化，移除跨页私有调用。
- 检查点 3：共享树三段式边界落地。

## 风险与回退办法

风险：

- 动作路由切换后，页面主动作行为退化。
- 树与页面之间出现双通道协作，造成边界回退。

回退办法：

- 如果某类动作无法一次性迁完，保留单一适配层并记录待删路径。
- 禁止为应急修复重新引入私有方法直接调用。
