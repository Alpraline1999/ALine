# Phase 0 Task 1

## 阶段

- Phase 0 / baseline-and-guardrails

## 对应方案

- `docs/refactor/01-phase-0-baseline-and-guardrails.md`

## 目标

- 固化主流程验证样本，避免后续阶段对“必须保持的行为”理解分叉。
- 把本轮重构的依赖禁令写成可机械检查的架构护栏。
- 明确每阶段只能执行窄范围验证，不做全量回归。

## 本任务范围

- 新增 Phase 0 任务文档。
- 新增架构护栏测试，覆盖：
  - `ui/**` 不得新增对 `project_manager`、`global_assets`、`extension_registry` 的直接导入。
  - `MainWindow` 不得新增对页面私有成员的直接访问。
  - `core/**` 不得新增对 `extensions/**` 的导入。
- 固化本轮主流程样本与阶段测试门槛。

## 主流程样本

1. 打开项目
2. 共享树选择
3. 数据导入
4. 加入绘图
5. 加入处理
6. 运行分析
7. 数字化导出
8. 保存项目

## 执行步骤

1. 建立本任务文件，记录边界、样本、验证和提交节点。
2. 扫描当前代码中的历史耦合，把现状固化为基线快照。
3. 为可立即收紧的规则添加硬禁令。
4. 为暂时不能一次性清零的历史耦合添加“禁止新增”基线测试。
5. 运行 Phase 0 窄测，只验证任务文档与架构护栏。
6. 按 `important-change-commit` 执行本阶段 `start` 提交。

## 验证

- `python -m unittest tests.test_refactor_guardrails`

## 提交节点

- `start`：任务文档与架构护栏测试落地。
- `checkpoint`：后续如果补充新的基线扫描项，再形成单独检查点提交。
- `end`：Phase 0 所有护栏、文档、验收项完成后提交。

## 约束

- 不修改业务算法和运行时主结构。
- 不提前拆 `ProjectManager`、页面状态或扩展系统。
- 不做大规模测试或全量 UI 回归。

## 完成判定

- 主流程样本清单明确且可复述。
- 护栏规则可通过测试机械检查。
- 本阶段验证命令可单独运行并通过。
